一、技术可行性
Q1 4meme 毕业 Token 可以直接在 Likwid 做多做空吗？
Blocker 问 Likwid

4meme 毕业后会在 PancakeSwap v3 上创建池子（fee=2500，0.25%），但 Likwid 的 LikwidMarginPosition 合约要求：

  • 存在对应的 Likwid 池子（Vault），而非 PancakeSwap 池
  • 池子有足够的 lend reserve + pair reserve（杠杆头寸所需资金）
  • 池子的 base LP fee 不低于合约配置的最低阈值（标准档位 3000/5000 是安全的，2500 未经验证）

核心问题：Likwid 是否愿意为 4meme 毕业 Token 创建并维护 Vault？还是需要我们自己 createPool() + 注入流动性？

如果完全 permissionless，我们能否在技术上独立完成而无需团队配合？

答案： 待确认
Q2 4meme 毕业池的 fee=2500 是否满足 Likwid 的 fee 阈值？
High 问 Likwid

Likwid 文档中 Integration Manual 给出的标准 fee 档位为 3000（0.3%）和 5000（0.5%）。4meme 毕业池使用 fee=2500（0.25%）。

如果 Likwid 的 LikwidMarginPosition 合约的 fee 阈值 > 2500，则该 Token 的池子对杠杆交易不可用。

需要向 Likwid 确认：合约当前配置的最低 fee 阈值是多少？或能否查询 marginMinFee（或类似）常量？

答案： 待确认（文档未公开具体数值）

二、合约安全性
Q4 合约的 Owner/Admin 可以执行哪些高权限操作？有无 pause 或升级风险？
Blocker 可自查

需要通过 BSCScan 或 Dedaub 查看 LikwidVault 和 LikwidMarginPosition 合约源码，确认：

  • 是否有 pause() 函数？Owner 是 EOA 还是多签？
  • 是否为 Proxy 合约（UUPS / Transparent Proxy）？能否被升级？

若合约可被 Owner 单方面 pause 且 Owner 是 EOA，属于高风险，需评估接受程度。

三、流动性现状
Q6 Likwid BSC 当前 TVL 和活跃池子数量是多少？
High 可自查

影响实际可用性的关键数据。可通过 DeFiLlama 和合约事件查询：

  • BSC Likwid TVL 当前值（DeFiLlama: likwid 协议页）

  • 最近 30 天 BSC 上 open position 成交量

若 TVL < $100K 且活跃池少于 5 个，杠杆功能对用户几乎不可用，集成价值大打折扣。

答案： 待查 DeFiLlama
Q7 若需要我们自行注入流动性，初始注入方案是什么？
Medium 问 Likwid

如果 Likwid 不为 4meme 毕业 Token 主动建 Vault，我们需要：

  • 调用 createPool() 创建 Likwid 池子
  • 向 Vault 注入 lend reserve（允许杠杆借贷）
  • 注入 pair reserve（提供对手方流动性）

问题：Binance Wallet 是否有预算为热门毕业 Token 主动注入初始流动性？谁来承担 IL 风险？

备选：与 Likwid 谈合作，由 Likwid 负责维护热门 Token 的 Vault，BW 负责流量引导。

答案： 待讨论（内部决策 + Likwid 合作意向）
